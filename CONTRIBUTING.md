# Contributing to Cold Email Playbook

Thank you for your interest in contributing to the Cold Email Playbook!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest improvements
- Include specific examples when possible
- Reference the relevant file(s) in your issue

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/improvement-name`)
3. Make your changes
4. Commit with clear, descriptive messages
5. Push to your fork
6. Open a Pull Request

### Content Guidelines

When contributing content:

- **Be specific**: Include concrete examples, not just theory
- **Be actionable**: Readers should be able to apply advice immediately
- **Cite sources**: Link to research or data when making claims
- **Stay current**: Cold email best practices evolve; note when advice may become dated
- **Smartlead focus**: Keep tooling references specific to Smartlead

### File Structure

```
best-practices/   → Guidelines and strategies (numbered for order)
context/          → Fill-in templates for company/ICP info
inputs/           → Editable campaign parameters
prompts/          → AI generation prompts
```

### Style Guide

- Use headers liberally for scanability
- Include code blocks for templates and examples
- Use tables for comparisons and reference data
- Keep individual sections focused and concise

## Questions?

Open an issue for any questions about contributing.
